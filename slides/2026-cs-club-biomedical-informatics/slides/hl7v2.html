HL7 v2

  • Pipe-delimited messages between systems
  • Very common in hospitals: ADT, orders, results
  • Ref: HL7 v2 product brief

MSH|^~\&|EHR|HOSP|LAB|HOSP|202602211530||ORU^R01|MSG00001|P|2.5.1
PID|1||123456^^^HOSP^MR||DOE^JANE||19900101|F
OBR|1||54321^LAB|88304^PATHOLOGY^LN
OBX|1|TX|NOTE||Specimen received.
  • v2 is pragmatic and widely deployed; many integrations still start here.
  • This example is an ORU^R01 "Observation Result" message: a lab system (or similar) sends results back to an EHR.
  • MSH is the message header (who/when/what type).
  • PID identifies the patient.
  • OBR is the order/request context for the result.
  • OBX carries one observation/result value.

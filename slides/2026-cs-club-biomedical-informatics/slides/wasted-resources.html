Wasted Resources

  • Every vendor re-writes importers, exporters, and API bridges
  • Each connector is one-off; M systems -> ~M^2 integrations
  • Version churn breaks glue code and creates a permanent integration tax
  • None of this differentiates your product
  • This is the same point as the proprietary-API bridge problem in other decks, but framed for CS students.
  • Open standards reduce duplication because one implementation can be reused across many partners.

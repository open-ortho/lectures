Prototype Tools

These can be useful to explore ideas, but are risky as clinical production software.

Common

  • Spreadsheets: Excel, Google Sheets, macros
  • Desktop databases: FileMaker Pro, Microsoft Access
  • No-code apps: Airtable, Smartsheet, PowerApps, AppSheet

Older

  • Visual Basic 6, VBA
  • Visual FoxPro, dBase, Paradox
  • Delphi, PowerBuilder, Lotus Notes forms
  • Non-shaming framing: these tools are fast, familiar, and sometimes all people have.
  • The risk shows up when they become systems of record, handle PHI, or drive clinical decisions.
  • Typical gaps: access control, audit trails, backups, validation, testing, upgrades, monitoring.
  • This is where standards and real software engineering practices pay off.

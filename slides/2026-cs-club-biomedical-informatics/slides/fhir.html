FHIR

  • Resources + REST API (plus messaging, docs, etc.)
  • Profiles: constrain FHIR for a specific use case
  • Spec: hl7.org/fhir

GET /fhir/Patient/123

{
  "resourceType": "Patient",
  "id": "123",
  "name": [{ "family": "Doe", "given": ["Jane"] }]
}
FHIR is the modern, web-friendly interoperability layer. In practice, you implement an IG, not "raw FHIR".
